Transitions vs Animations

Similarities

  1. Specify which CSS properties to listen for changes.
  2. Set timing (easing) functions to alter the rate of going from a one property value to another.
  3. Specify a duration to control how long the animation or transition will take.
  4. Programmatically listen to animation and transition-specific events that you can then do with as you wish.
  5. Visualize CSS property changes.

Differences

Animations and transitions show their differences when it comes to how you trigger them to play, whether they loop easily, how complicated of a transition you can define, how formal you must be in being able to use them, and how well they play with JavaScript.

CSS Animations

The key to being able to tell an animation from a transition is to check the number of times the animation is being repeated. If it is once, then it is a transition. If you are using a animation-iteration count, then it is an animation.
